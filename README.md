# 🤖 资产表现监控 (Systemic Asset Monitor)

> **核心哲学**：本系统是一个彻底封闭的“黑盒”监控引擎，只监控各类资产本周表现及技术指标，为**跨市场固定比例战略资产配置 (All-Weather + Barbell Strategy)** 提供数据支持。它采用“全域降维时空锁定”技术过滤噪音，剥离日常情绪波动，仅提供最纯粹的微观动量信号（多头/空头趋势、52周极值）与周度绝对方向（🔴涨 / 🟢跌），辅助执行机械化的纪律性再平衡。

## 1. 系统架构与双端运行机制

* **部署环境**：完全托管于 GitHub Actions 云端，本地无需开机。
* **触发机制**：Cron 定时任务。**每周六北京时间上午 10:00** 自动运行。
* **双端异构输出引擎**：
  * **📨 SMTP 原生邮件直连**：系统通过 Python 原生封装 HTML/CSS，直连个人邮箱（支持 QQ/163/126 等）。配合微信的“QQ邮箱提醒”功能，实现无广告、极简 APP 视觉的秒级弹窗。
  * **🌐 专属 Web 阅读器**：基于 GitHub Pages 部署的 Apple 极简风网页端（`report-viewer.html`）。内置强力反缓存机制与防墙（GFW）兜底逻辑，可随时通过下拉菜单优雅查阅历史归档。
* **时空锁定算法**：引擎内置“日线降维与周五强制锚定”机制。无论脚本在周末何时触发，系统会自动切除未闭合的数据幻影，强行对齐到**最近一个日历周五的 23:59:59**，确保与正规行情软件的周线绝对同频。
* **多源数据穿透**：无缝对接 Yahoo Finance、东方财富 (Eastmoney)、腾讯财经，以及 Coinbase 机构级公开 API，自动绕过云端 IP 封锁。

## 2. 核心配置库：`portfolio.csv` 维护指南 (⚠️最重要)

系统的所有监控目标及展示流向，全部由仓库根目录下的 `portfolio.csv` 控制。
**如需增减持仓、修改名称、调整监控池，只需在 GitHub 网页端点击 `portfolio.csv` -> 点击右上角“铅笔”图标编辑 -> 提交保存即可。**

### 📝 表头格式（必须严格遵守）

文件第一行必须永远是（全部小写英文，半角逗号，**没有多余空格**）：

```csv
ticker,name,type,pool,market
```

### 🎛️ 核心参数控制说明

| 列名 | 功能描述 | 填写规范摘要 |
| :--- | :--- | :--- |
| **`ticker`** | 资产代码 | 严格遵循各市场代码规范（详见下方规则）。 |
| **`name`** | 自定义显示名称 | **严禁包含英文逗号`,`**。尽量精简（如：`标普500ETF`）。 |
| **`type`** | 底层通道路由 | 股票/场内ETF:`yf` \| 场外基金:`jj` \| 加密货币:`crypto` |
| **`pool`** | 核心/备选分流 | `core`（核心持仓） \| `watch`（备选观察） \| `core watch`（双重属性） |
| **`market`**| 宏观物理隔离 | **决定排版归类**，仅限填写指定的 7 大类（见下文）。 |

**📌 核心字段详细填写规则：**

* **`ticker` (代码规范)**：
  * **A股 / 场内 ETF**：上交所末尾加 `.SS`（如 `600900.SS`）；深交所末尾加 `.SZ`（如 `159928.SZ`）。
  * **港股**：原生代码末尾加 `.HK`（如 `00700.HK`）。
  * **美股**：直接写交易所原生代码（特殊：伯克希尔B股写为 `BRK-B`）。
  * **场外公募基金**：纯 6 位数字代码，**千万不要带任何前缀或字母**（如 `005287`）。
  * **加密货币**：币种全大写，必须带 `/USDT` 交易对后缀（如 `BTC/USDT`）。
* **`market` (资产宏观属性)**：
  必须**一字不差**地从以下 7 个类别中选择并填入，系统将据此分配 Emoji 标签进行物理隔离排版：
  `A股`、`港股`、`美股`、`其他海外市场`、`国内外债券`、`大宗商品及贵金属`、`加密货币`。

## 3. 日常部署与手动测试

### ⚙️ 核心环境变量配置
初次使用或更换发件邮箱时，必须在 GitHub 仓库的 `Settings -> Secrets and variables -> Actions` 中配置以下两个变量：
* `EMAIL_USER`：你的邮箱地址（例如 `xxx@126.com` 或 `xxx@qq.com`）。
* `EMAIL_PASS`：该邮箱开启 SMTP 服务后获取的**授权码**（绝对不是网页登录密码）。

### 🚀 手动点火巡检
若修改了标的，或想临时查看实时数据，可随时打破周六定时进行手动巡检：
1. 在 GitHub 仓库点击顶部 **Actions** 标签页。
2. 左侧点击 **Systemic Asset Monitor**。
3. 点击右侧 **Run workflow** 按钮。
4. 等待 1 分钟。网页端阅读器将自动更新，同时微信/邮箱会收到最新推送。

## 4. 常见故障排查 (FTA)

如果你遇到数据不同步或报错，99% 是由以下原因导致：

* **❌ 邮件推送失败: (535, b'Error: authentication failed')**
  * **病因**：SMTP 认证失败。
  * **解法**：检查 `EMAIL_PASS` 是否填成了登录密码。必须去邮箱设置里生成专用的 SMTP **授权码**。
* **⚠️ 网页阅读器显示“数据同步失败 / 尝试强连模式”**
  * **病因 1**：你的 GitHub 仓库是私有 (Private) 状态，导致开放 API 拒绝前端访问。
    * **解法**：在 `Settings -> General` 底部将其改为 **Public**。
  * **病因 2**：国内网络波动，屏蔽了 `api.github.com`。
    * **解法**：无需惊慌，阅读器已内置抗压兜底机制，此时虽然看不到历史下拉菜单，但页面会自动盲猜并强行加载当天生成的最新版报告。
* **⚠️ 报告底部的“异常信息”警示框出现标的**
  * **病因**：代码填写错误、该标的退市/停牌，或遇到极低概率的 API 连环封锁。
  * **解法**：检查 `portfolio.csv` 里的 `ticker` 是否拼错。若无错，忽略即可，系统会自动将其隔离，不会导致引擎崩溃。
* **🖥️ 网页阅读器刷新后仍是旧数据**
  * **病因**：GitHub Pages 静态服务器部署需要 1~3 分钟的物理延迟。
  * **解法**：Actions 跑完后请耐心等待 2 分钟再刷新网页。

## 5. 系统纪律与信号执行

本系统为“最大化者”人格设计。在周末查阅报告时，请严格遵守“杠铃策略”与机械化准则，杜绝情绪化主观预判：

* **📊 核心持仓 (Core)**
  * 仅提供资产网格的宏观方向感知。在未触发固定比例偏离阈值（如某资产大涨导致仓位严重超标）时，**不进行任何主观干预**，继续机械执行定投。
  * **纪律铁律**：我计划不考虑经济周期，不同大类资产一直按固定比例进行定投和再平衡。
* **🧭 趋势系统 (5周/20周动量)**
  * **🟢 多头**：当前收盘价 5 周均线 ≥ 20 周均线。代表中期右侧趋势稳健。
  * **🔴 空头**：当前收盘价 5 周均线 ＜ 20 周均线。代表中期趋势承压或破位。
  * **执行逻辑**：针对处于激进端的观察期资产，由空转多可作为建仓发令枪；由多转空必须严格审视风控，机械截断亏损。
* **🚨 极值标签 (新高 / 新低)**
  * **[新高] / [新低]**：标的突破过去 52 周（一年）的最高点或最低点。
  * **执行逻辑**：作为动量衰竭或主升浪确认的辅助参考。不接飞刀，不追顶点。
* **👀 备选观察 (Watch)**
  * 此板块中的行业 ETF 与宽基指数（如科创50、道琼斯等）仅作为宏观侦察兵。它们触发异动，仅作为大类资产轮动的预警，绝不代表立刻调仓核心资产。
