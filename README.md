## 1. 系统架构与运行机制

* **部署环境**：完全托管于 GitHub Actions 云端，本地无需开机。
* **触发机制**：Cron 定时任务。**每周六北京时间上午 10:00** 自动运行并推送`方糖`公众号。
* **数据引擎**：
* `yfinance`: 负责提取 A股、港股、美股及场内 ETF 数据。
* `akshare`: 负责提取国内场外公募基金（净值转周线）。
* `yfinance`: 负责提取加密货币数据（直连 Binance）。


* **推送通道**：Server酱 Webhook，直达微信端，按涨跌幅降序排列。
* **引擎极速模式**：已彻底砍掉所有外部名称字典拉取接口，系统 100% 信任本地 CSV 配置的名称，启动速度极快且零报错。

## 2. 核心配置库：`portfolio.csv` 维护指南 (⚠️最重要)

系统的所有监控目标全部由仓库根目录下的 `portfolio.csv` 控制。
**如需要增减持仓、修改名称，只需在 GitHub 网页端点击 `portfolio.csv` -> 点击右上角“铅笔”图标编辑 -> 提交保存即可。**

### 📝 表头格式（必须严格遵守）

文件第一行必须永远是：

```csv
ticker,name,type

```

*(注意：所有的逗号必须是**英文半角逗号**)*

### 🔍 三大资产类型 (`type`) 输入规范

| 资产类型            | `type` 标识 | `ticker` 代码填写规则                               | 示例 |
| ------------------- | ----------- | --------------------------------------------------- | ---- |
| **股票 / 场内 ETF** | `yf`        | **美股**：直接写原生代码。                          |      |
| **基金**            | `yf`        | 纯 6 位数字代码。千万不要带 `jj` 等任何前缀或字母。 |      |
| **加密货币**        | `crypto`    | 币种全大写，带 `/USDT` 交易对后缀。                 |      |

**A股**：`6`或`5`开头加 `.SS`(上交所)，`0` `1` `3`开头加 `.SZ`(深交所)。

如：股票：`600519.SS`、ETF：`159928.SZ`

**港股**：代码加 `.HK` 后缀。

如：`00700.HK`

**美股**：直接写交易所中代码。*(特殊：伯克希尔B股必须写为 `BRK-B`)* 

如：`NVDA`

**加密货币**：币种全大写，带 `/USDT` 交易对后缀。

如：`BTC/USDT`、`ETH/USDT` 

**场外公募基金**：纯 6 位数字代码。千万不要带 `jj` 等任何前缀或字母。

如： `005287`、`100050`

### 🏷️ 名称列 (`name`) 的绝对避坑指南

因为系统是“哑巴哨兵”，它会**绝对服从**你在这里填写的任何文字。填写时请务必注意：

1. **绝不能包含英文逗号 `,**`：CSV 文件是以英文逗号作为列分隔符的。如果你写了 `Meta Platforms, Inc.`，系统会认为这里被切成了两列，导致全盘报错！必须把名称里的逗号删掉，写成 `Meta Platforms Inc.`。
2. **尽量简短**：为了在手机微信端显示美观（不折行），名称尽量控制在 8 个汉字以内，例如将“易方达标普500指数(QDII)联接A”简写为“标普500易方达”。
3. **不可留空**：哪怕不知道叫什么，也要把代码再抄一遍填在 name 列，绝不能空着。

**正确的 CSV 样本片段：**

```csv
ticker,name,type
600900.SS,长江电力,yf
NVDA,英伟达,yf
0700.HK,腾讯控股,yf
005287,海富通创业板,jj
BTC/USDT,比特币,crypto
BRK-B,伯克希尔B,yf

```

## 3. 日常维护与手动测试

在增减了标的，或者想临时查看当天净值表现时，你可以随时打破周六的定时，进行手动巡检：

1. 登录 GitHub，进入 `mechanical-asset-monitor` 仓库。
2. 点击顶部的 **Actions** 标签页。
3. 在左侧菜单点击 **Systemic Asset Monitor** 工作流。
4. 点击右侧出现的 **Run workflow** 下拉菜单 -> 点击绿色 **Run workflow** 按钮。
5. 等待 30-60 秒，微信即可收到最新报告。

## 4. 常见故障排查 (FTA)

如果你在微信没有收到报告，或者 Actions 中亮起了红叉（❌），请点进报错日志。99% 的问题出在以下两点：

* **报错包含 `KeyError`, `IndexError` 或 `Columns**`
* **病因**：你最近修改 `portfolio.csv` 时破坏了格式。
* **解法**：检查是否有空行？检查标的名称里是不是不小心敲了英文逗号 `,`？检查表头是不是还是 `ticker,name,type`？


* **未收到推送，但 GitHub 显示绿勾（✅）**
* **病因**：Server酱的通信 Token 异常。
* **解法**：去 Server酱官网查看本月推送额度是否耗尽，或重新获取 SendKey，去 GitHub 仓库的 `Settings -> Secrets and variables -> Actions` 中更新 `SERVERCHAN_KEY`。


* **个别标的没出现在报告里**
* **病因**：代码写错了（例如深交所写成了 `.SS`），或者 Yahoo Finance / 基金接口暂时性网络抽风。
* **解法**：核对代码后缀。若是接口网络问题无需理会，系统带有静默容错机制，某个标的获取失败会直接跳过，不会导致整体系统崩溃。

## 5. 系统纪律与信号执行

在周末收到这份带有 🔴/🟢 的微信简报时，严格遵守你的“最大化者”准则与以下逻辑，避免过度交易：

* **常规阵地（📊 资产涨跌幅榜）**
* 仅提供宏观方向感知。在未触发你的固定比例偏离阈值（例如某资产大涨导致仓位占比超标 5%）时，**不进行任何操作**，继续执行定投。


* **极端敞口管理（🎯 核心阵地异动）**
* **触发 ✅ 金叉确立 (5周上穿20周)**：代表右侧趋势确认。对于处于“杠铃策略”极度激进端的资产（或一直等待建仓的底仓、定投持仓），执行买入/加仓指令。
* **触发 ⚠️ 死叉离场 (5周下穿20周)**：代表中期趋势破位。机械性执行风控减仓，无论当时消息面如何，必须截断亏损或锁定利润。
* **触发 🚀/🩸 一年期极值**：作为动量（Momentum）和情绪衰竭的辅助参考指标，结合当前仓位比重进行再平衡评估，不接飞刀。



> *“我计划不考虑经济周期，不同大类资产一直按这个比例进行定投和再平衡。” —— 纪律是唯一能穿越周期的武器。*
