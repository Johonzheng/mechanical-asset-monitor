# 🤖 资产表现监控 (Systemic Asset Monitor)

> **核心哲学**：本系统是一个彻底封闭的“黑盒”监控引擎，只监控各类资产本周表现及技术指标，为**跨市场固定比例战略资产配置**提供数据支持。它采用“全域降维时空锁定”技术过滤噪音，仅提供最纯粹的微观动量信号（5周/20周均线交叉、52周极值）与周度绝对方向（🔴涨 / 🟢跌），辅助执行机械化的纪律性再平衡。

## 1. 系统架构与运行机制

* **部署环境**：完全托管于 GitHub Actions 云端，本地无需开机。
* **触发机制**：Cron 定时任务。**每周六北京时间上午 10:00** 自动运行并推送 Server酱 微信通道。
* **时空锁定算法**：引擎内置“日线降维与周五强制锚定”机制。无论周末何时触发运行，系统会自动切除本周末的未闭合数据幻影，强行对齐到**最近一个日历周五的 23:59:59**，确保与所有正规行情软件的周线绝对同频。
* **多源异构数据引擎**：
  * **股票/场内 ETF (`yf`)**：主战线使用 Yahoo Finance，无缝搭载东方财富 (Eastmoney) 与腾讯财经双模容错底层，自动绕过云端 IP 封锁，智能处理美股代码变异。
  * **场外公募基金 (`jj`)**：直连东方财富天天基金网，提取精准单位净值。
  * **加密货币 (`crypto`)**：彻底抛弃传统爬虫，直连美国最大合规交易所 **Coinbase Pro** 的机构级公开 API，实现 7x24 小时抗封锁极速穿透。

## 2. 核心配置库：`portfolio.csv` 维护指南 (⚠️最重要)

系统的所有监控目标及展示流向，全部由仓库根目录下的 `portfolio.csv` 控制。
**如需增减持仓、修改名称、调整监控池，只需在 GitHub 网页端点击 `portfolio.csv` -> 点击右上角“铅笔”图标编辑 -> 提交保存即可。**

### 📝 表头格式（必须严格遵守）

文件第一行必须永远是（全部小写英文，半角逗号）：

```csv
ticker,name,type,active,pool
```

### 🎛️ 核心参数控制说明

| 列名 | 功能描述 | 填写规范摘要 |
| :--- | :--- | :--- |
| **`ticker`** | 资产代码 | 严格遵循各市场代码规范（详见下方规则）。 |
| **`name`** | 自定义显示名称 | 系统**绝对服从**此列。**严禁包含英文逗号`,`**。尽量精简（如：`标普500ETF`）。 |
| **`type`** | 底层通道路由 | 股票/ETF:`yf` \| 场外基金:`jj` \| 加密货币:`crypto` |
| **`active`** | 监控启停开关 | 填 **`y`** 开启监控。<br>填 **`n`** 则系统直接忽略该行，无需删除代码即可拉黑。 |
| **`pool`** | 智能分流标签 | 决定该标的在报告中的去向（支持多标签并存，见下文）。 |

**📌 `ticker` 代码详细填写规则（极其重要）：**

* **A股 / 场内 ETF**：
  * **上交所**：`6` 或 `5` 开头，末尾加 `.SS`。如：`600519.SS`、`512690.SS`。
  * **深交所**：`0`、`1`、`3` 开头，末尾加 `.SZ`。如：`159928.SZ`、`000895.SZ`。
* **港股**：原生代码末尾加 `.HK`。如：`00700.HK`、`03033.HK`。
* **美股**：直接写交易所原生代码。*(特殊：伯克希尔B股必须写为 `BRK-B`，系统会自动兼容)*。如：`NVDA`、`AAPL`。
* **场外公募基金**：纯 6 位数字代码，**千万不要带 `jj` 等任何前缀或字母**。如：`005287`、`100050`。
* **加密货币**：币种全大写，必须带 `/USDT` 交易对后缀。如：`BTC/USDT`、`ETH/USDT`。

### 🔍 Pool 智能分流与宏观参照系

本系统支持将核心交易资产与宏观参照基准进行物理隔离，在 `pool` 列中填写以下标签控制去向：

1. **`core`**（核心持仓）：将出现在 **“🎯 核心持仓异动”** 及底部的 **“📊 核心持仓涨跌幅龙虎榜”** 中。
2. **`watch`**（备选库）：将独立出现在 **“🔍 备选自选异动”** 中，**不会**混入你的持仓龙虎榜，避免污染视线。
3. **`core watch`**（双重属性）：带空格或逗号分隔。既在自选异动中作为“大盘/行业宏观参照”进行信号预警，又参与总盘涨跌幅计价。

**正确的 CSV 样本片段：**

```csv
ticker,name,type,active,pool
600900.SS,长江电力,yf,y,core
BTC/USDT,比特币,crypto,y,core
159928.SZ,消费ETF汇添富（消费行业参照）,yf,y,core watch
399300.SZ,沪深300参照,yf,y,watch
TLT,iShares20年美债,yf,n,core
```
*(注：TLT 的 active 为 `n`，它将完全不参与本次计算；沪深 300 仅作为大盘信号参考，不进龙虎榜。)*

## 3. 日常维护与手动测试

在增减了标的，或者想临时查看本周实时净值表现时，你可以随时打破周六的定时，进行手动巡检：

1. 登录 GitHub，进入本仓库。
2. 点击顶部的 **Actions** 标签页。
3. 在左侧菜单点击 **Systemic Asset Monitor** 工作流。
4. 点击右侧出现的 **Run workflow** 下拉菜单 -> 点击绿色 **Run workflow** 按钮。
5. 等待 30-60 秒，微信即可收到最新对齐的格式化周报。

## 4. 常见故障排查 (FTA)

如果你在微信没有收到报告，或者 Actions 中亮起了红叉（❌），请点进报错日志。99% 的问题出在以下两点：

* **报错包含 `KeyError`, `IndexError` 或列未对齐**
  * **病因**：你最近修改 `portfolio.csv` 时破坏了格式。
  * **解法**：检查是否有空行？检查标的 `name` 里是不是不小心敲了英文逗号 `,`？检查表头是否遗漏了列？
* **未收到推送，但 GitHub 显示绿勾（✅）**
  * **病因**：Server酱的通信 Token 异常或额度耗尽。
  * **解法**：去 Server酱官网查看额度，或重新获取 SendKey 更新至仓库的 `Settings -> Secrets and variables -> Actions` 中。
* **个别标的没出现在报告的龙虎榜里**
  * **病因 1**：你把它的 `pool` 设置成了仅为 `watch`，它已经被隔离到自选雷达区了。
  * **病因 2**：你把 `active` 设置成了 `n`。
  * **病因 3**：极小概率的网络震荡。系统带有静默容错机制，某个标的全部备用通道告破时会直接跳过，列入底部的“盲区公示”，绝不会导致整体崩溃。

## 5. 系统纪律与信号执行

在周末收到这份报告时，请严格遵守“最大化者”与“杠铃策略”准则，避免情绪化过度交易：

* **📊 核心持仓龙虎榜**
  * 仅提供资产网格的宏观方向感知。在未触发固定比例偏离阈值（如某资产大涨导致仓位严重超标）时，**不进行任何干预操作**，继续机械执行定投。
* **🎯 核心持仓异动监控**
  * **触发 ✅ 金叉确立 (5周上穿20周)**：代表右侧趋势确认。针对处于激进端的观察期资产，执行建仓/加仓指令。
  * **触发 ⚠️ 死叉离场 (5周下穿20周)**：代表中期趋势破位。机械性执行风控减仓，无视利好消息，必须截断亏损或锁定利润。
  * **触发 🚀/🩸 一年期极值**：作为动量（Momentum）衰竭的辅助参考指标，不接飞刀，不追顶点。
* **🔍 备选自选异动监控（宏观雷达）**
  * 此板块中的行业 ETF 与宽基指数（如科创50、纳斯达克、半导体等）仅作为前置侦察兵。当某个行业参照触发金叉/新高时，可作为轮动底仓配置的发令枪，但不代表立刻调仓核心资产。

> *"我计划不考虑经济周期，不同大类资产一直按这个比例进行定投和再平衡。" —— 纪律是唯一能穿越周期的武器。*
